# Colorscheme Analysis

Deep analysis of six reference schemes, scored for **long-form reading** and
**UI token reuse** — not for syntax highlighting, which is what most of them
were originally built for. That distinction matters: a color that's perfect as
a code keyword (an accent glanced at for a fraction of a second) can fail as
body text or a link you stare at for minutes.

All contrast ratios below are **computed**, not quoted — generated by
`tools/color.py` against each scheme's canonical hex values
(`tools/schemes.json`). Run `python3 tools/analyze.py` to regenerate.

WCAG shorthand used in tables:

- **AAA** ≥ 7:1 · **AA** ≥ 4.5:1 · **AA-large** ≥ 3:1 (only valid for ≥18px text or UI) · **fail** < 3:1

A quick note on reading the "fail" rows: surface/overlay colors are _supposed_
to be near-invisible against the bg (that's their job — 1.1:1 is correct for an
`overlay` on `base`). A "fail" only matters when the color is meant to carry
**text or an icon**.

## TL;DR scorecard

| Scheme          | Light          | Dark         | Body text   | Accents-as-UI      | Reading verdict                                                           |
| --------------- | -------------- | ------------ | ----------- | ------------------ | ------------------------------------------------------------------------- |
| **Flexoki**     | ✅ excellent   | ✅ excellent | AAA / AAA   | all AA+ both modes | **Best all-rounder.** Designed for ink-on-paper, every accent is UI-safe. |
| **Solarized**   | ✅ good        | ✅ good      | 4.7–4.99    | AA-large only      | Scientifically tuned _symmetry_, but deliberately low-contrast body text. |
| **Gruvbox**     | ✅ good        | ✅ excellent | AAA         | mixed              | Warm, high-contrast, opinionated. Light accents weaker than dark.         |
| **Tokyo Night** | ⚠️ weak light  | ✅ excellent | AAA (dark)  | AA+ (dark)         | Dark is gorgeous and accessible. Light ("Day") is an afterthought.        |
| **Catppuccin**  | ⚠️ weak light  | ✅ excellent | AAA (Mocha) | **fail (Latte)**   | Mocha is great; Latte's pastels collapse on light bg.                     |
| **Everforest**  | ⚠️ syntax-only | ✅ good      | AA          | **fail (light)**   | Soothing, but accents are tuned as low-contrast syntax, not UI.           |

**My pick to learn from:** Flexoki for structure and discipline, Gruvbox for
warmth, Solarized for the symmetry idea. Catppuccin/Tokyo Night/Everforest are
better mood references than accessibility references.

## 1. Flexoki — by Steph Ango (Obsidian CEO)

> "An inky color scheme for prose and code." Designed _first_ for reading, with
> a base gray ramp tuned on the OKLab lightness scale and accents chosen to hold
> contrast in both modes.

### Core palette

| Role           | Light           | Dark            |
| -------------- | --------------- | --------------- |
| bg             | `#fffcf0` paper | `#100f0f` black |
| surface        | `#f2f0e5`       | `#1c1b1a`       |
| ui / border    | `#e6e4d9`       | `#282726`       |
| text           | `#100f0f`       | `#cecdc3`       |
| text-2 (muted) | `#6f6e69`       | `#878580`       |
| accent (blue)  | `#205ea6`       | `#4385be`       |

### What makes it distinctive

Flexoki is the only scheme here engineered around **perceptual lightness from
the start**. The base ramp (`paper` → `black`, with 50/100/.../900 steps) is
spaced on OKLab L\*, so tints feel evenly stepped. Crucially, it ships **two
accent ramps**: a darker "600" set for light mode and a brighter "400" set for
dark mode — the same hue identity, re-tuned per background. That's exactly the
"not just inverted" principle done right.

### Light/dark approach

Not inverted — _retuned_. Light text is near-black (`#100f0f`, 18.6:1), dark
text is a warm off-white (`#cecdc3`, 12:1, deliberately **not** pure white to
avoid halation). Each accent has a dedicated value per mode.

### Contrast (computed)

| Color          | Light vs paper  | Dark vs black   |
| -------------- | --------------- | --------------- |
| text           | **18.62:1** AAA | **11.98:1** AAA |
| text-2 (muted) | 4.97:1 AA       | 5.19:1 AA       |
| red            | 6.23:1 AA       | 4.42:1 AA-large |
| orange         | 4.69:1 AA       | 5.77:1 AA       |
| yellow         | 3.39:1 AA-large | 8.08:1 AAA      |
| green          | 4.39:1 AA-large | 6.12:1 AA       |
| blue           | 6.36:1 AA       | 4.86:1 AA       |
| purple         | 7.58:1 AAA      | 5.39:1 AA       |
| magenta        | 6.53:1 AA       | 5.14:1 AA       |

### Strengths / weaknesses for reading

- **+** Best-in-class. Every accent clears AA in _both_ modes — almost unheard of.
- **+** Muted text (`tx2`) lands at ~5:1 in both modes: visibly secondary but still readable. This is the hardest thing to get right and Flexoki nails it.
- **+** Warm paper bg + warm off-black ink = genuinely paper-like, low glare.
- **−** Light yellow/green sit at AA-large — fine for headings/large, marginal for small colored text. (Yellow is always the problem child; see §6.)
- **−** Aesthetically _restrained_ — some find it too plain. That's a feature for prose, a limitation if you want personality.

**Verdict:** The template. This is what I'd base the system's discipline on.

## 2. Solarized — by Ethan Schoonover

> The original "scientifically-grounded" scheme (2011). Built in CIELAB with a
> fixed set of 16 colors and a **symmetric** design: the same 8 accent colors
> work on both the dark and light backgrounds, and the four bg/fg tones are
> mirror images across the L\* midpoint.

### Core palette

The signature trick: one set of accents, two backgrounds.

| Role                     | Value                                                                                                                                   |
| ------------------------ | --------------------------------------------------------------------------------------------------------------------------------------- |
| base03 (darkest bg)      | `#002b36`                                                                                                                               |
| base02 (dark surface)    | `#073642`                                                                                                                               |
| base00 / base0 (body)    | `#657b83` / `#839496`                                                                                                                   |
| base2 / base3 (light bg) | `#eee8d5` / `#fdf6e3`                                                                                                                   |
| accents (both modes)     | yellow `#b58900`, orange `#cb4b16`, red `#dc322f`, magenta `#d33682`, violet `#6c71c5`, blue `#268bd2`, cyan `#2aa198`, green `#859900` |

### What makes it distinctive

The **symmetry**. Schoonover fixed each accent's L* so it reads acceptably on
*either\* background. The four-step mono ramp (base03/02/01/00 ↔ base0/1/2/3) is
selected so that "dark body text" and "light body text" have matching relative
contrast. It's the most intellectually rigorous scheme here.

### Contrast (computed)

| Color      | Dark (vs base03)    | Light (vs base3)           |
| ---------- | ------------------- | -------------------------- |
| body text  | base0 **4.75:1** AA | base00 **4.13:1** AA-large |
| emphasized | base1 5.61:1 AA     | base01 4.99:1 AA           |
| blue       | 4.08:1 AA-large     | 3.41:1 AA-large            |
| cyan       | 4.75:1 AA           | 2.93:1 **fail**            |
| green      | 4.69:1 AA           | 2.97:1 **fail**            |
| yellow     | 4.68:1 AA           | 2.98:1 **fail**            |
| red        | 3.25:1 AA-large     | 4.29:1 AA-large            |
| violet     | 3.44:1 AA-large     | 4.05:1 AA-large            |

### Strengths / weaknesses for reading

- **+** The symmetry idea is genuinely useful: a _shared accent set_ is a real maintenance win (see Proposal C).
- **+** Low-contrast-by-design body text (4.1–4.75:1) reduces glare for long sessions — Schoonover's whole thesis.
- **−** That same low contrast is the catch: it _just barely_ passes AA in dark and **misses AA for normal text in light** (base00 = 4.13). The aesthetic is "muted," which many people read as "washed out."
- **−** The shared-accent symmetry doesn't actually hold for accessibility: cyan/green/yellow that pass on dark **fail on light**. The colors look balanced but aren't equally legible.

**Verdict:** Borrow the _philosophy_ (symmetric ramp, shared accents), not the
exact values. Its contrast is a 2011 compromise that WCAG has since outgrown.

## 3. Gruvbox — by Pavel Pertsev (morhetz)

> Warm, retro, high-contrast. Heavy earth tones over a brown-gray bg. The most
> "personality-forward" scheme here.

### Core palette (dark medium)

| Role           | Value                                                                                                                |
| -------------- | -------------------------------------------------------------------------------------------------------------------- |
| bg0            | `#282828`                                                                                                            |
| bg (soft/hard) | `#32302f` / `#1d2021`                                                                                                |
| surface        | `#3c3836`                                                                                                            |
| fg             | `#ebdbb2`                                                                                                            |
| gray (muted)   | `#928374`                                                                                                            |
| accents        | red `#fb4934`, green `#b8bb26`, yellow `#fabd2f`, blue `#83a598`, purple `#d3869b`, aqua `#8ec07c`, orange `#fe8019` |

### What makes it distinctive

The **warmth**. The bg isn't neutral gray — it's a warm `oklch(27.7% … 90)`
brown-black, and the fg is a creamy `#ebdbb2`. Accents are highly saturated and
earthy (mustard yellow, brick red). It feels like a 1970s terminal in the best
way. It also ships explicit hard/medium/soft bg contrast variants — a nice idea.

### Contrast (computed)

| Color        | Dark (vs bg0)   | Light (vs bg)   |
| ------------ | --------------- | --------------- |
| fg           | **10.75:1** AAA | 10.22:1 AAA     |
| gray (muted) | 4.02:1 AA-large | 3.24:1 AA-large |
| red          | 4.29:1 AA-large | 7.60:1 AAA      |
| green        | 7.14:1 AAA      | 4.29:1 AA-large |
| yellow       | 8.69:1 AAA      | 3.33:1 AA-large |
| blue         | 5.48:1 AA       | 5.82:1 AA       |
| orange       | 5.84:1 AA       | 5.40:1 AA       |
| purple       | 5.37:1 AA       | 5.94:1 AA       |

### Strengths / weaknesses for reading

- **+** Excellent fg contrast (10:1+) in both modes — never a legibility problem for body.
- **+** Dark mode accents are strong; warm palette is easy on the eyes at night.
- **−** The muted "gray" only hits ~3.2–4:1 — too weak for AA body, fine for de-emphasized large text.
- **−** Asymmetry: bright accents tuned for dark (red 4.29 dark) get _re-darkened_ for light mode (red 7.60 light) — Gruvbox maintains two separate hand-tuned palettes. Great results, but high maintenance.
- **−** Saturation is high; some accents (bright red `#fb4934`) can vibrate against the dark bg.

**Verdict:** The warmth reference. Gruvbox proves a warm, high-personality
palette can still be highly legible. Steal the _temperature_, lower the saturation.

## 4. Tokyo Night — by enkia

> Cool, blue-leaning, the VS Code darling. Inspired by Tokyo's neon-at-night
> palette. Dark-first; the light ("Day") variant is clearly secondary.

### Core palette (Night)

| Role            | Value                                                                                                                |
| --------------- | -------------------------------------------------------------------------------------------------------------------- |
| bg              | `#1a1b26`                                                                                                            |
| bg_dark         | `#16161e`                                                                                                            |
| bg_highlight    | `#292e42`                                                                                                            |
| fg              | `#c0caf5`                                                                                                            |
| comment (muted) | `#565f89`                                                                                                            |
| accents         | blue `#7aa2f7`, cyan `#7dcfff`, purple `#bb9af7`, green `#9ece6a`, red `#f7768e`, orange `#ff9e64`, yellow `#e0af68` |

### What makes it distinctive

A genuinely **cool** identity — the bg is `oklch(22.6% 0.021 280)`, a desaturated
blue-violet, and the fg is a soft periwinkle. The accents are pastel-bright but
not neon. It pairs beautifully with the cool-dark direction the existing Journey
tokens already lean toward (`hsl(225 18% 11%)`).

### Contrast (computed)

| Color           | Night (vs bg)   | Day (vs bg)     |
| --------------- | --------------- | --------------- |
| fg              | **10.59:1** AAA | 4.52:1 AA       |
| comment (muted) | 2.76:1 **fail** | 2.54:1 **fail** |
| blue            | 6.79:1 AA       | 3.11:1 AA-large |
| cyan            | 9.96:1 AAA      | 4.26:1 AA-large |
| purple          | 7.39:1 AAA      | 3.33:1 AA-large |
| green           | 9.35:1 AAA      | 4.04:1 AA-large |
| red             | 6.46:1 AA       | 3.01:1 AA-large |
| yellow          | 8.55:1 AAA      | 3.75:1 AA-large |

### Strengths / weaknesses for reading

- **+** Dark mode is excellent: fg 10.6:1, every accent AA or better, mostly AAA. Hard to beat for night reading.
- **+** Cool palette aligns with the existing dark tokens — a natural reference for Proposal B.
- **−** The `comment` color (2.76:1) **fails** even in dark mode. This is the classic dark-mode trap: muted text too dark. If you adopt it, lighten comments.
- **−** "Day" is weak across the board — only fg clears AA, every accent is AA-large at best. Don't use Tokyo Night as a _light_ reference.

**Verdict:** The cool-dark reference. Take the Night palette's bg/fg/accent
relationships; ignore Day and fix the comment color.

## 5. Catppuccin — by the Catppuccin org

> Pastel, modern, _extremely_ popular (4 flavors: Latte light; Frappé /
> Macchiato / Mocha dark). Soft, low-saturation, friendly.

### Core palette

| Role             | Mocha (dark)          | Latte (light) |
| ---------------- | --------------------- | ------------- |
| base (bg)        | `#1e1e2e`             | `#eff1f5`     |
| mantle / crust   | `#181825` / `#11111b` | `#e6e9ef` / — |
| surface0         | `#313244`             | `#ccd0da`     |
| text             | `#cdd6f4`             | `#4c4f69`     |
| subtext0 (muted) | `#a6adc8`             | `#6c6f85`     |
| accent (mauve)   | `#cba6f7`             | `#8839ef`     |

### What makes it distinctive

A meticulously **systematized pastel** identity with 26 named colors per flavor
and a whole brand around it. Mocha's bg is a soft blue-violet `oklch(24.3% 0.03 284)`
— cousin to Tokyo Night but warmer and lighter. The aesthetic is "cozy."

### Contrast (computed)

| Color            | Mocha (vs base) | Latte (vs base) |
| ---------------- | --------------- | --------------- |
| text             | **11.34:1** AAA | 7.06:1 AAA      |
| subtext0 (muted) | 7.37:1 AAA      | 4.37:1 AA-large |
| mauve            | 8.07:1 AAA      | 4.79:1 AA       |
| blue             | 7.79:1 AAA      | 4.34:1 AA-large |
| red              | 7.08:1 AAA      | 4.80:1 AA       |
| green            | 11.03:1 AAA     | 2.96:1 **fail** |
| yellow           | 12.91:1 AAA     | 2.31:1 **fail** |
| peach            | 9.27:1 AAA      | 2.64:1 **fail** |
| teal             | 11.01:1 AAA     | 3.31:1 AA-large |

### Strengths / weaknesses for reading

- **+** **Mocha is superb** — text 11.3:1, every accent AAA. One of the best dark reading palettes that exists.
- **+** Beautifully systematized; great naming to copy (base/mantle/crust/surface/overlay/text).
- **−** **Latte is a trap.** On a light bg the same pastel philosophy produces accents that _fail AA_: green 2.96, yellow 2.31, peach 2.64. Pastels need a _dark_ background to gain contrast; on light they wash out. This is the single most important lesson in the whole document.
- **−** Even Latte's muted `subtext0` (4.37) misses AA for normal text.

**Verdict:** Mocha = top-tier dark reference and the naming scheme to adopt.
Latte = a cautionary tale about pastels on light backgrounds.

## 6. Everforest — by sainnhe

> Green-based, low-contrast-by-design, marketed for "protecting your eyes."
> Soft, foresty, comfortable. Built explicitly as a _soothing_ scheme.

### Core palette (medium)

| Role           | Dark                  | Light                 |
| -------------- | --------------------- | --------------------- |
| bg0            | `#2d353b`             | `#fdf6e3`             |
| bg_dim / bg1   | `#232a2e` / `#343f44` | `#f2efdf` / `#f4f0d9` |
| fg             | `#d3c6aa`             | `#5c6a72`             |
| gray (muted)   | `#859289`             | `#939f91`             |
| accent (green) | `#a7c080`             | `#8da101`             |

### What makes it distinctive

The **low-contrast-on-purpose** philosophy and the green cast through both bg and
accents. The light bg `#fdf6e3` is the _same_ warm paper Solarized uses. It's the
most explicitly "comfort over punch" scheme here.

### Contrast (computed)

| Color        | Dark (vs bg0)   | Light (vs bg0)  |
| ------------ | --------------- | --------------- |
| fg           | **7.38:1** AAA  | 5.18:1 AA       |
| gray (muted) | 3.84:1 AA-large | 2.56:1 **fail** |
| red          | 4.55:1 AA       | 3.04:1 AA-large |
| orange       | 5.41:1 AA       | 2.48:1 **fail** |
| yellow       | 6.84:1 AA       | 2.12:1 **fail** |
| green        | 6.23:1 AA       | 2.69:1 **fail** |
| aqua         | 5.89:1 AA       | 2.79:1 **fail** |
| blue         | 5.74:1 AA       | 3.13:1 AA-large |
| purple       | 5.40:1 AA       | 2.83:1 **fail** |

### Strengths / weaknesses for reading

- **+** Dark mode is comfortable and decent: fg 7.38, all accents AA.
- **+** The warm-paper light bg is lovely, and fg (5.18) is fine for body.
- **−** **Light-mode accents are a bloodbath** — 6 of 8 fail AA. Like Catppuccin Latte, the "soft" tuning that works on dark falls apart on light. These accents are designed as _syntax highlight_ (glanceable code tokens), not UI text.
- **−** Muted gray fails AA in both practical uses.

**Verdict:** The "comfort/soothing" mood reference and a second warm-paper data
point. But its accent values are syntax-only — do not reuse for links/buttons.

## Cross-scheme lessons (what I'm taking into the proposals)

1. **Pastels need dark backgrounds.** Catppuccin Latte and Everforest light both prove that soft accents that shine on dark _fail_ on light. On light bg, accents must be **darker and more saturated**, not lighter. (Flexoki and Gruvbox both do exactly this — separate darker accent ramp for light.)

2. **Two accent ramps beat one.** Solarized's "shared accent set" is elegant but its accessibility doesn't survive both backgrounds (cyan/green/yellow fail on light). Flexoki/Gruvbox's two-ramp approach wins. Proposal C will keep _hue identity_ shared but retune _lightness_ per mode — the real lesson of "not just inverted."

3. **Muted/secondary text is where everyone fails.** Tokyo Night `comment` (2.76, fails even in dark!), Catppuccin Latte `subtext0` (4.37), Everforest `gray` (2.56). Target **≥ 4.5:1 for any muted text** that carries real content; reserve sub-4.5 only for truly decorative text. Flexoki's ~5:1 in both modes is the bar.

4. **Off-black, off-white — never pure.** Every good dark scheme uses an off-white fg (Flexoki `#cecdc3`, Gruvbox `#ebdbb2`, Tokyo Night `#c0caf5`) and an off-black bg, to avoid halation. Pure `#000`/`#fff` appears in none of them.

5. **Yellow is structurally hard on light.** Yellow's high intrinsic luminance means it _always_ struggles for contrast on light bg (Flexoki 3.39 AA-large, Gruvbox 3.33, Solarized 2.98 fail). For a `warning` role on light, push yellow toward amber/ochre (lower L, toward orange) — never lemon.

6. **Warm bg + cool dark is a defensible split.** Gruvbox/Flexoki/Solarized/Everforest all use warm paper for light; Tokyo Night/Catppuccin Mocha use cool for dark. The existing Journey tokens (warm light, cool dark) are already on the right track — Proposal C formalizes it.

## Where this diverges from the general research (and why)

This is the _applied_ layer. The reusable, general design-system knowledge lives
separately in the Hermes research set (`~/Works/Hermes/research{,-site}/design-system`),
and its color synthesis (`personal-color-palette.md`) lands on a different default:
**Flexoki-light + Everforest-dark, "warm everywhere," with an accent that adapts
hue per mode** (cool blue on light, warm green on dark).

This repo deliberately goes elsewhere on three points. None of them contradict the
general knowledge on _facts_ — the contrast math, WCAG thresholds, OKLCH authoring,
and "retune don't invert" are identical. They're taste/scope calls:

1. **Cool blue-black dark, not warm green-gray.** The general synthesis prefers
   warmth in both modes (Everforest `#2D353B`). This system keeps the existing
   Journey cool-dark (`#171a21`) for a deliberate _temperature contrast_ between
   modes and a one-step migration from what's already shipped. Trade-off is owned
   in `proposals/proposal-C.md` → "Honest trade-offs."

2. **One re-lit purple, not an adaptive blue/green.** The general synthesis swaps
   accent _hue_ per mode (what "pops" differs by background temperature). This
   system holds a single purple identity (~286–291°) and moves only lightness —
   the Solarized "shared accent" idea done accessibly (§2, lesson #2). It favors
   brand consistency over per-mode pop.

3. **CSS custom properties only, not DTCG JSON + a Global tier.** The general
   tokens guidance prescribes a portable DTCG JSON layer (Global → Semantic) for
   Style Dictionary. This system stays single-tier semantic CSS on purpose —
   `AGENTS.md` defers build-time token transformers "for now." Revisit if the
   system ever needs to output to non-web targets (iOS/Android/Figma).

If any of these stop being deliberate, this is the note to delete.
